Description

What's the problem?
Given items with weights and values, and a knapsack with limited capacity, choose items to maximize total value without exceeding capacity.

Which algorithm are we using?
Dynamic Programming (DP). We solve smaller subproblems and reuse their results to build the final answer efficiently.

When do we use 0/1 Knapsack?
Use it when:
  • Each item can be taken at most once (0 or 1 decision)
  • You need to maximize value under a constraint (like weight, cost, time, etc.)
  • Greedy approaches don’t guarantee the optimal answer
If wᵢ ≤ w: K[i,w] = max(K[i−1,w], vᵢ + K[i−1,w−wᵢ]) Else: K[i,w] = K[i−1,w]

Greedy Counterexample and Why DP Wins

Dynamic Programming: include / exclude recurrence

Exclude branch: K[i-1,w]
Include branch: v_i + K[i-1,w-w_i] if w_i ≤ w
Decision rule: store the larger of the two values.
Why it works: DP checks both choices at every state instead of committing early to one ratio-based choice.

How to understand the chart

Why the runtime label does not change

Plain-English explanation

Think of the DP table like a grid. The algorithm still moves across the whole grid, so the main runtime stays O(nW).
Some cases look slower only because more cells need the extra "include this item" check. That changes the amount of work inside the loop, not the overall runtime category.
When the table is done, the algorithm walks backward to recover the answer. That traceback part is much smaller: O(n).

Keep this in mind

These explanations match the actual version shown in this visualizer: a full 2D DP table is stored.
You may learn another knapsack version later that uses only O(W) space. That is real, but it is not the one being visualized here.
For brute force, the important idea is simple: the number of possible subsets explodes as 2^n, which is why it becomes impractical much faster.

Why that matters

A greedy ratio order is a good first thought, but it is not a correctness proof for 0/1 knapsack.
DP compares include and exclude at every relevant state, so it can undo an attractive-looking local choice.
After the table is finished, traceback performs a search through the table to recover the chosen subset.
